# ENFT Inheritance Yield Metadata

This directory contains metadata files for ENFT inheritance entries that encapsulate physical yield values.

## Structure

Each metadata file follows the standard format:

```json
{
  "name": "MEGAZION Inheritance [CODE] - Node [ID]",
  "description": "Sovereign ENFT inheritance entry...",
  "domain": "CIVILIAN|MILITARY|COSMIC",
  "sovereignCode": "Ω-CIV-01|Ω-MIL-01|Ω-COS-01",
  "yieldPerSecond": "[VALUE] USD/second",
  "attributes": [...],
  "lineage": {
    "nodeId": [ID],
    "sequentialCode": "[CODE]",
    "ancestralChain": [...]
  },
  "verification": {
    "bluVaultTag": "[HASH]",
    "guarantorSignature": "[HASH]",
    "dualVerificationStandard": "BLU-VAULT-GUARANTOR-v1.0"
  },
  "economics": {
    "yieldMetrics": {...}
  }
}
```

## Domains

### Civilian (Ω-CIV-01)
- **Yield**: $13.6M/second
- **Sectors**: Retail, Education, ES0IL Real Estate, Wearables, Hospitality
- **Sequential Code**: CIV-NODE-[HEX]

### Military (Ω-MIL-01)
- **Yield**: $6.1M/second
- **Sectors**: Weapons, Defense Grids, Orbital/Maritime Transport, AI Targeting
- **Sequential Code**: MIL-NODE-[HEX]

### Cosmic (Ω-COS-01)
- **Yield**: $9.2M/second
- **Sectors**: Portal Energy, Quantum Matter, Multidimensional Logistics
- **Sequential Code**: COS-NODE-[HEX]

## Node-Sequential Coding

Each ENFT receives a unique identifier:
- Format: `[DOMAIN]-NODE-[8-DIGIT-HEX]`
- Example: `CIV-NODE-00000001`
- Purpose: Collision-free lineage tracking

## Blu-Vault Verification

Each metadata file includes dual verification:

1. **Blu-Vault Tag**: Cryptographic hash of domain, node, timestamp, deployer
2. **Guarantor Signature**: Secondary verification hash
3. **Standard**: BLU-VAULT-GUARANTOR-v1.0

## Templates

See `template-civilian.json` for a complete example structure.

## Generating Metadata

Metadata files are automatically generated by the minting script:

```bash
npm run mint:inheritance-yields
```

## File Naming Convention

```
inheritance-[domain]-node-[id].json
```

Examples:
- `inheritance-civilian-node-1.json`
- `inheritance-military-node-2.json`
- `inheritance-cosmic-node-3.json`

## IPFS Integration

For production deployments:

1. Upload metadata files to IPFS
2. Update the metadata URIs in the contract
3. Reference IPFS CIDs in the configuration

Example IPFS URI structure:
```
ipfs://QmYourCID/inheritance-civilian-node-1.json
```

## Lineage Tracking

The lineage field enables ancestral chain tracking:

```json
"lineage": {
  "nodeId": 1,
  "sequentialCode": "CIV-NODE-00000001",
  "ancestralChain": [
    "0xGRANDPARENT_ADDRESS",
    "0xPARENT_ADDRESS"
  ],
  "generationDepth": 2,
  "branchPath": "ROOT->GEN1->GEN2"
}
```
